# ScenarioSeed Positioning

## Core Insight

The underserved problem is not fake data generation. It is fake business-state generation.

Developers can already generate names, emails, products, and rows. What still takes manual work is building coherent states such as:

- a trial account expiring in two days
- a high-value customer with a failed invoice
- an active workspace above usage quota
- an order with payment failure and reserved inventory
- a refunded order with finance, fulfillment, and support views aligned

These states are the fixtures that make product development, QA, demos, and AI coding-agent debugging feel real.

## High-Star Wedge

The project should win by being:

- instantly understandable: "Faker creates fake fields. ScenarioSeed creates fake businesses."
- instantly runnable: no API key required for the core path
- visually demonstrable: before/after screenshots and generated scenario briefs
- framework-adjacent: start with Prisma, then expand to Drizzle, Supabase, Rails, SQL
- agent-native: let AI coding agents request named states instead of asking users to handwrite fixtures

## What To Avoid

- generic synthetic data claims
- evaluation or benchmarking positioning
- requiring an LLM key on first run
- trying to infer every business perfectly
- competing head-on with mature fake data libraries
